mvc architecture

m stands for model
v stands for view
c stands for controller

models are the actual java script code that we type to do a particular task
it might be a function or variable or algorithm

views are actually the ui front end part connected to the modules via the controllers
whatever the user sees interacts performs happens via the view
eg body tag

controllers act as a bridge between the model and view
the index.html connects to the app.js v

scope are the variables that hold the data inside the controllers
